Introduction

 

The Graphic Pump automatically moves copies of images from the Internet to your Screen Saver, Pocket PC, Photo Wallet, or Digital Picture Frame.

 

It is a utility that sits on the system tray and pumps images to these devices, applying formatting along the way.  Images can be pumped to these devices on a pre-determined schedule, or on demand. 

 

The application ships as freeware.  Feel free to distribute it to anyone you choose.

 

For the latest version, news, or to make feature suggestions visit:

 

http://www.graphicpump.com

 

History and Uses

 

The history behind the Graphic Pump is simple. It started as a simple goal. I wanted a way to have my photo albums displayed on a variety of devices, ranging from a Pocket PC to a Screen Saver to a digital picture frame.  Sometimes the photos came from multiple locations, sometimes I simply came across photos that were interesting, and sometimes I wanted to get photos and images from the Internet that were updated periodically.  I wanted to be able to update my photos at a central place and have them be updated on all these devices. Finally, I wanted to be able to update the photos in my albums frequently, but not have to repeat the process of both manipulating my photos and moving my photos to all my devices and locations.

 

I not only wanted this capability for myself, but also wanted to setup the application for my relatives, with me controlling the photos they would see remotely.

 

Out of this simple idea came the Graphic Pump. 

 

The Graphic Pump’s main purpose is to load, format, and save JPG files to a new location.

 

There are three main concepts in the application:

 

  1. Formats Profiles

 

A user typically deals with a fixed number of devices and fixed number of desired formats for JPG files on these devices.  This is handled in the Graphic Pump through a set of global named Format Profiles.  Each Format Profile stores the various settings for the final saved JPG file (e.g. size, compression, etc).

 

  1. Pumping an Image on Demand

 

In its simplest form, the Graphic Pump can take an individual image and Pump (save) it to a new location.  Think of this as a “Common Save” dialog that remembers all the various JPG formats you may wish to apply to the image.

 

  1. Tasks

 

The program allows you to schedule "tasks" that are either run on demand, or repeated on a schedule.  A task has four main properties:

 

Source for the images - This can be images on a hard disk or images off of the Internet. It can also be an album file containing pointers to images on the hard disk or off the Internet.

 

Schedule for running the task - This can be on demand, hourly, daily, weekly, or monthly

 

Formatting for the images - This allows you to optimize the images for a particular device, and apply some simple margin, padding, and rotation along the way. This formatting uses the same Format Profiles discussed above.

 

Destination for the images - This is where you want the images (and other output) to go

 

Installing and Running the Application

 

Installing the application is simple.  A simple setup.exe program is run.  This installs the Graphic Pump application, the Graphic Pump Screen Saver, and several supporting libraries.

 

By default the application places a shortcut into the Startup program group. This allows the Graphic Pump to run once the system is booted.  You can remove this shortcut or place it somewhere else, but running the application at startup is necessary to run tasks on a schedule. You can also simply run the executable (GraphicPump.exe) from Windows if desired.

 

Once the application is running, an icon appears in the system tray.  The icon is a small "pump" with the letter "G" in it.

 

Clicking on this icon will launch the main configuration dialog. Right clicking on the icon brings up a menu that allows you to bring up the main configuration dialog or exit the application.

Format Settings

 

Format Settings Dialog

 

Target Size: Width

 

This is the desired width for the resulting images in pixels. If no resize rules are applied (grow, shrink, or rotate), this value is ignored.

 

Target Size: Height

 

This is the desired height for the resulting images in pixels.  If no resize rules are applied (grow, shrink, or rotate), this option is ignored.

 

Target Size: Thumbnail

 

This option indicates that a thumbnail should be generated inside the resulting output file.  Generating a thumbnail increases the size of the target file, and not all output devices take advantage of this extra data.  However, devices that do utilize this thumbnail will benefit.

 

Target Size: Thumb Width

 

This is the width of the thumbnail.

 

Target Size: Thumb Height

 

This is the height of the thumbnail.

 

Resizing: Shrink to Size

 

This tells the Graphic Pump that the image should be shrunk to fit in the target width and height.  The aspect ratio will be preserved when doing the resize.  In other words, if the source image is 200 pixels wide by 400 pixels high, and the desired format is 100 pixels wide by 100 pixels wide, the final image will be 50 pixels wide by 100 pixels high.  This leaves the aspect ratio of the image alone and shrinks to the smallest width to display the image.

 

Resizing: Grow to Size

 

This tells the Graphic Pump that the image should grow to fit in the target width and height.  The aspect ratio will be preserved when doing the resize.

 

Rotate: Rotate to Size

 

This tells the Graphic Pump to rotate the image 90 degrees to maximize the size of the image given the desired width and height.  This option only makes sense when the resulting image will be displayed on a device that is easily rotated for display (such as a pocket PC). 

 

For example, if the desired image size is 640 pixels wide by 480 pixels high, and the image to pump is 100 pixels wide by 500 pixels high, the image is rotated to allow the bigger dimension of the source image to take 640 pixels in the final image.

 

Rotate: Direction

 

This tells the Graphic Pump to rotate clockwise or counter clockwise if a rotation is necessary.

 

JPEG Settings: Compression

 

This is the amount to compress the final image.  The higher the compression, the lower the quality of the image and the less disk space the image takes.  The lower the compression, the higher the quality of the image and the more disk space the image takes.

 

Padding: Pad to Size

 

This tells the Graphic Pump that the final image should be padded with pixels to match the desired width and height when either growing or shrinking the image.  If the image is not grown or shrunk to the desired size, padding options are not available.

 

In the example above, the image would be padded with 50 pixels horizontally to make the final image 100 pixels wide by 100 pixels high.

 

Padding: Color

 

This is the color that should be used to pad the image to the desired size. To change the padding color, click “Change” next to the pad color and select a new color.

 

Padding: Vertical

 

This is the vertical (up/down) alignment of the resized image in the final padded image of the full size.  Options are top, center, and bottom.

 

Padding: Horizontal

 

This is the horizontal (left/right) alignment of the resized image in the final padded image of the full size. Options are left, center, and right. In the example above, left alignment would yield the extra 50 pixels (of pad color) on the right (the image is aligned to the left), center would yield 25 pixels on the right and 25 pixels on the left, and right would yield the extra 50 pixels on the left (the image is aligned to the right).

 

Margins

 

This tells the Graphic Pump to place margins around the image.  This can give the image an appearance similar to a photograph.

 

Margins: Top

 

This is the number of pixels to place on the top of the image when margins are specified.

 

Margins: Left

 

This is the number of pixels to place on the left of the image when the margins are specified.

 

Margins: Right

 

This is the number of pixels to place on the right of the image when the margins are specified.

 

Margins: Bottom

 

This is the number of pixels to place on the bottom of the image when the margins are specified.

 

Margins: Color

 

This is the color to use for the margins.  To change the margin color, click “Change” next to the margin color.

 

OK

 

This accepts changes made to the formatting.  Click the "OK" button to accept changes for the task.

 

Cancel

 

This cancels changes made to the formatting.  Click the "Cancel" button to cancel changes for the task.

Destination

 

The destination dialog allows you to specify where the resulting images should be written, and how they should be written.

 

There are two ways images can be written.  They can be written to a directory in the file system or they can be written to a Digi-Frame digital picture frame through the serial port.

 

With these simple ways of writing images, there are many ways that the images can be used. 

 

They can be used as your personal image screen saver. By writing the images to a directory and using the included Graphic Pump Screen Saver (which reads images from a directory), the images are shown whenever your PC is idle.

 

They can be displayed on a pocket PC.  By writing the images to the sync directory of a pocket PC as a mini-web site, you can have a mini photo album that travels with you.

 

They can be displayed on a digital picture frame or other hardware device.  By writing directly to the Digi-Frame digital picture frame, the images are automatically rendered on the frame.  Other brands of picture frames and other hardware devices such as the Sony picture frame or the VideoChip Photo Wallet can also the target destination by writing a directory of images onto a memory stick or compact flash card.

 

There are several options with each destination type.

 

The destination is specified using the drop down on the left of the destination dialog.

This option will write all the images into a directory on the hard disk.

This tells the Graphic Pump to delete all files that may have been written by a previous execution of the Graphic Pump.  Checking this option will cause the Graphic Pump to delete any files that match the file name specified.

File Name

 

This is a template name for the name of the file that the Graphic Pump should generate for the resulting images.  This file name must contain the characters %i%.  The Graphic pump will substitute a number for the %i% characters at run time.

 

In other words, if there are three images in the task and the file name is file%i%.jpg, three files named file0000.jpg, file0001.jpg, and file0002.jpg will be written into the destination directory.

 

If Delete Contents is checked, any file named file*.jpg where * is anything will be deleted when the task is run.

Graphic Pump Screen Saver

 

A simple screen saver is included and installed along with the application.  This is mostly so there is something interesting to do with images processed by the Graphic Pump out of the box.  Not everyone has a Pocket PC or a digital picture frame and would like to display their images somewhere.

 

The screen saver is simply a bonus and is included with all copies of the application in unrestricted form. There are better "shareware" image screen savers out there.  By all means use them instead of the one included here.

 

The options and functionality for the Graphic Pump Screen Saver are simple.  The screen saver can be configured with an individual image, or a directory of images.  It will rotate through the images (if more than one), or display the image (if only one) with a background color and float around the screen.

 

To use the Graphic Pump Screen Saver, right click on the Windows Desktop and select "Properties" from the popup menu.  The "Display" dialog should appear.  Select the "Screen Saver" tab.  Select "Graphic Pump Screen Saver" from the "Screen Saver" Dropdown.  Click the "Settings" button and the "Graphic Pump Screen Saver" dialog should appear.  To see what it will look like, click the "Preview" button.

 

Screen Saver Dialog

 

Image/Path - This is either a directory containing jpg images or an individual jpg image for display.  Either type in the name of a directory or a jpg file, or click the "Browse" button to find one on your computer.

 

Background Color - This is the color the rest of the screen should display in the background.  To change the color, click the "Change" button and select a new color.

 

Movement: Speed - This is the speed of movement in milliseconds (1000 = 1 second), or in other words how often the location of the image should be updated. Experiment with different settings here and see what works best on your computer and to your liking. To have a fixed static image that doesn't move input 0.

 

Movement: Horizontal Delta - This is the number of pixels the image should travel horizontally for every move. To not have the image move horizontally, enter 0.

 

Movement: Vertical Delta - This is the number of pixels the image should travel vertically for every move. To not have the image move vertically, enter 0.

 

Flip: Time - This is the time of flipping images in milliseconds (1000 = 1 second), or in other words how often the image should remain visible before switching to the next image.
